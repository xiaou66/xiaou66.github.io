# B 树

B 树 (balanced Tree) 是一种平衡的多路搜索树，多用于文件系统、数据库的实现

# 特点

  1. 1 个节点可以存储超过 2 个元素、可以拥有超过 2 个子节点
  2. 拥有二叉搜索树的一些性质
  3. 平衡，每个节点的所有子树高度一致
  4. B 树中的阶指的是一个结点最多拥有的结点个数

# M 阶 B 树的性质

假设一个结点存储的元素个数是 m

  1. 根结点 1≤x≤m−11 \le x \le m-11≤x≤m−1
  2. 非根结点 ⌈m/2⌉−1≤x≤m−1\left \lceil m/2 \right \rceil -1 \le x \le m-1⌈m/2⌉−1≤x≤m−1

如果有子结点，子结点个数 y=x+1y = x + 1y=x+1

  1. 根结点: 2≤y≤m2 \le y \le m2≤y≤m

  2. 非根结点 ⌈m/2⌉≤x≤m\left \lceil m/2 \right \rceil \le x \le m⌈m/2⌉≤x≤m

比如

  • m=3,2≤y≤3m = 3, 2 \le y \le 3m=3,2≤y≤3 可以称为 2-3 树
  • m=4,2≤y≤4m = 4, 2 \le y \le 4m=4,2≤y≤4 可以称为 2-3-4 树

# B 树与 BST 树关系

  1. B 树 和 二叉搜索树，在逻辑上是等价的
  2. 多代结点合并，可以获得一个超级结点
    • 2 代合并的超级节点，最多拥有 4 个子节点（至少是 4 阶 B 树）
    • 3 代合并的超级节点，最多拥有 8 个子节点（至少是 8 阶 B 树）
    • n 代合并的超级节点，最多拥有 2n2^n2n 个子节点（ 至少是 2n2^n2n 阶 B 树）

m 阶 B 树，最多需要 log⁡2m\log_{2}{m\\}log2​m 代合并

# 搜索

跟二叉搜索树的搜索类似

  1. 先在节点内部从小到大开始搜索元素
  2. 如果命中，搜索结束
  3. 如果未命中，再去对应的子节点中搜索元素，重复步骤 1

# 添加

新添加的元素必定是添加到叶子节点

  1. 正常插入
    • 插入 「95」

  1. 当插入时叶子结点的元素个数超过限制时需要进行「上溢」操作

    • 插入 「101」

    如果上溢结点最中间元素的位置为 k

    1. 将 k 位置的元素向上与父结点合并
    2. 将[0，k−1][0，k - 1][0，k−1] 和 [k+1,m−1][k + 1, m - 1][k+1,m−1] 位置的元素分裂成 2 个子结点
      • 这两个子结点的元素个数，必然都不会低于最低限制 ⌈m/2⌉−1\left \lceil m/2 \right \rceil - 1⌈m/2⌉−1

# 删除

# 叶子结点

  1. 假如需要删除的元素在叶子节点中，那么直接删除即可

# 非叶子结点

  1. 先找到前驱或后继元素，覆盖所需删除元素的值

  2. 再把前驱或后继元素删除

    非叶子节点的前驱或后继元素，必定在叶子节点中

    真正的删除元素都是发生在叶子节点中

# 删除造成下溢

叶子节点被删掉一个元素后，元素个数可能会低于最低限制 (≥⌈m/2⌉−1\ge\left \lceil m/2 \right \rceil - 1≥⌈m/2⌉−1)

# 解决方案

  • 如果下溢节点临近的兄弟节点，有至少 ⌈m/2⌉\left \lceil m/2 \right \rceil⌈m/2⌉ 个元素，可以向其借一个元素

    1. 将父节点的元素 b 插入到下溢节点的 0 位置（最小位置）
    2. 用兄弟节点的元素 a（最大的元素）替代父节点的元素 b

  • 如果下溢节点临近的兄弟节点，只有 ⌈m/2⌉−1\left \lceil m/2 \right \rceil - 1⌈m/2⌉−1 个元素

    1. 将父节点的元素 b 挪下来跟左右子节点进行合并
    2. 合并后的节点元素个数等于⌈m/2⌉+⌈m/2⌉−2≤m−1\left \lceil m/2 \right \rceil + \left \lceil m/2 \right \rceil - 2 \le m − 1⌈m/2⌉+⌈m/2⌉−2≤m−1
    3. 这个操作可能会导致父节点下溢，依然按照上述方法解决，下溢现象可能会一直往上传播

# 4 阶 B 树的性质

  1. 所有节点能存储的元素个数 x ：1≤x≤31 \le x \le 31≤x≤3
  2. 所有非叶子节点的子节点个数 y ：2≤y≤42 \le y \le 42≤y≤4
